# First 5 Beta Users Plan

## Who to Invite First
Invite users who:
- are active Amazon sellers with weekly optimization routines,
- can test consistently for 7 days,
- can provide structured, candid feedback,
- are comfortable with beta limitations.

## Ideal Beta User Profile
- 50-500 ASIN seller account (but will test with only 1-3 ASINs in beta).
- Uses at least one analytics tool already.
- Understands rank, buy box, and availability signals.
- Can share screenshots and reproducible bug reports.

## 7-Day Beta Structure
- Day 0: Access + onboarding + first actions.
- Day 1: First real workflow run.
- Day 2: First feedback pulse.
- Day 3-4: Repeat workflows and consistency checks.
- Day 5: Value and trust review.
- Day 6: Follow-up fixes/workarounds if needed.
- Day 7: Decision interview and next-step ask.

## Messaging Timeline
### Day 0
- Welcome message.
- Share test script.
- Set expectation: test with 1-3 ASINs only.
- Share known limitations upfront.

### Day 2
- Ask for first friction points.
- Capture one most useful module and one blocker.
- Confirm if they could complete the full workflow once.

### Day 5
- Ask value questions:
  - What saved time?
  - What is still missing?
  - What data do they trust vs not trust yet?

### Day 7
- Final review:
  - overall score,
  - willingness to continue,
  - willingness to pay (and why),
  - must-fix list before wider rollout.

## What Success Means (First 5 Users)
- At least 4 of 5 users complete full script end-to-end.
- No P0/P1 unresolved blockers in core modules.
- At least 3 users report recurring weekly value.
- Clear top 3 roadmap priorities from feedback.

## What to Monitor Manually
- Setup friction and connection failures.
- Sync and refresh consistency (keyword/BSR/buybox/pincode).
- Alert quality (false positives and misses).
- Time-to-value from login to first useful insight.
- User trust in output for decision-making.

## When to Stop Onboarding More Users
Pause onboarding if any of these occur:
- Any unresolved P0 in core flow.
- Multiple users blocked by same P1 issue.
- Daily monitoring shows unstable refresh reliability.
- Alert quality creates repeated false-critical confusion.

## Move from 5 Users to 20 Users When
Proceed only when:
- 7 consecutive days without unresolved P0/P1,
- core workflow success rate is stable,
- feedback from first 5 confirms repeat usage value,
- team has capacity for support and daily monitoring at 20-user scale.
